1. General Workflow

The Harvester is a lightweight, Ruby-based engine that turns any FAIR resource URL into structured metadata that the FAIR Champion tests can consume. It follows a predictable, reproducible sequence shown in the diagram above.

STEP 01

Input

A landing-page URL, PID, or any resolvable identifier is provided to the harvester.

STEP 02

Content Negotiation

The harvester requests the resource in multiple formats (JSON-LD, Turtle, RDF/XML, HTML, plain JSON, etc.) using HTTP Accept headers.

STEP 03

Parsing & Extraction

Metadata is parsed according to its format. RDF graphs, JSON-LD, Microdata, and embedded schemas are all supported.

STEP 04

Predicate Matching

Specific predicates and patterns are searched to locate the data record, self-identifier, license, and other key pieces of information.

The entire process is deterministic and fully transparent. Every decision and discovered triple is logged for debugging.

How it finds the pointer to the data record

The harvester scans the parsed metadata for any of the following predicates (in order of preference). The first match becomes the pointer to the actual data.

http://www.w3.org/ns/ldp#contains
http://xmlns.com/foaf/0.1/primaryTopic
http://purl.obolibrary.org/obo/IAO_0000136
http://schema.org/mainEntity
http://schema.org/distribution
http://schema.org/contentUrl
http://www.w3.org/ns/dcat#distribution
http://www.w3.org/ns/dcat#downloadURL
http://semanticscience.org/resource/SIO_000332
https://purl.obolibrary.org/obo/IAO_0000136
…and their https equivalents

How it finds the self-identifier of the metadata record

http://purl.org/dc/elements/1.1/identifier http://purl.org/dc/terms/identifier http://schema.org/identifier https://purl.org/dc/terms/identifier https://schema.org/identifier

The value of the first matching predicate is treated as the canonical identifier of the metadata record itself.

How it detects the type of GUID / PID

The harvester applies a series of regular expressions (in priority order) to classify the identifier:

Type Pattern (Ruby regex)
inchi^\w{14}-\w{10}-\w$
doi^10\.\d{4,9}/[-._;()/:A-Z0-9]+$ (case-insensitive)
